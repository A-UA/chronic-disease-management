# Patient Service (Phase 2.2) CRUD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the complex, real-world CRUD logic within the newly scaffolded Patient Microservice for both Java and NestJS. Ensure soft-RLS applies everywhere.

**Domains:**
1. Health Metrics (`health_metrics`)
2. Patient Family Links (`patient_family_links`)

---

### Task 1: Health Metrics Entity & Controller (Java & NestJS)

**Goal:** Provide endpoints to Create, Read, and List health metrics for patients based on soft RLS boundaries.

- [ ] **Step 1: NestJS Entity & DTOs**
  - Create `HealthMetricEntity` mapping to `health_metrics` table. Provide columns: `id`, `tenant_id`, `org_id`, `patient_id`, `metric_type`, `metric_value` (JSON/string), `recorded_at`.
  - Create controller endpoints taking `IdentityPayload` and filtering by `allowedOrgIds`.
  
- [ ] **Step 2: Java Entity & Repository**
  - Create `HealthMetricEntity.java` with JPA decorators.
  - Create `HealthMetricRepository.java` extending `JpaRepository` with custom contextual queries.
  - Create Controller endpoints mapped under `/api/v1/health-metrics`.

- [ ] **Step 3: Commit**
  - Add and commit the Health Metrics functionality.

---

### Task 2: Patient Family Links & Assignees

**Goal:** Implement endpoints for associating family members to patients, allowing them cross-organizational view access based on rules.

- [ ] **Step 1: Define Entities**
  - `PatientFamilyLinkEntity` (tenant, org, patient_id, family_user_id, relationship).

- [ ] **Step 2: Dual Implementation**
  - NestJS Service layer handling nested queries taking into account RLS boundaries.
  - Java Service layer utilizing Repository `@Query` filters.

- [ ] **Step 3: Validation and Error Handling**
  - Verify that a parent/child link respects tenant boundaries.

- [ ] **Step 4: Commit**
  - Add and commit the Family Links functionality.

---

### Task 3: API Gateway Passthrough

- [ ] **Step 1: Update API Gateways**
  - Expose `/api/v1/health-metrics/**` and `/api/v1/family-links/**` explicitly mirroring payload proxy behaviors from Phase 2.1.
  - Apply Auth filters properly.
